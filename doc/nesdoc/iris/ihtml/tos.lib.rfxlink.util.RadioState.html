Interface: tos.lib.rfxlink.util.RadioState

interface RadioState

Commands
command uint8_t getChannel() Returns the currently selected channel.
command error_t setChannel(uint8_t channel) Sets the current channel.
command error_t standby() The same as the turnOff command, except it is not as deep sleep, and it is quicker to recover from this state.
command error_t turnOff() Moves to radio into sleep state with the lowest power consumption but highest wakeup time.
command error_t turnOn() Goes into receive state.

Events
event void done() This event is signaled exactly once for each sucessfully posted state transition and setChannel command when it is completed.

Commands - Details

getChannel

command uint8_t getChannel()

Returns the currently selected channel.

setChannel

command error_t setChannel(uint8_t channel)

Sets the current channel. Returns EBUSY if the stack is unable to change the channel this time (some other operation is in progress) SUCCESS otherwise.

standby

command error_t standby()

The same as the turnOff command, except it is not as deep sleep, and it is quicker to recover from this state.

turnOff

command error_t turnOff()

Moves to radio into sleep state with the lowest power consumption but highest wakeup time. The radio cannot send or receive in this state and releases all access to shared resources (e.g. SPI bus).

turnOn

command error_t turnOn()

Goes into receive state. The radio continuously receive messages and able to transmit.

Events - Details

done

event void done()

This event is signaled exactly once for each sucessfully posted state transition and setChannel command when it is completed.
